# Story 10.1: Property Management Authority Page

## User Story

**As a** homeowner dealing with Bold Property Management
**I want** a comprehensive page explaining their role, authority, and limitations
**So that** I understand what they can and cannot do on behalf of the HOA

## Description

Create a dedicated `/management` page that provides a complete picture of Bold Property Management's role, delegated authority, and limitations. This page becomes the authoritative resource for understanding the management relationship and sets clear expectations for homeowners.

## Acceptance Criteria

- [ ] New page at `/management` route
- [ ] Page header clearly identifies Bold Property Management as the subject
- [ ] Sections for: Overview, What Bold Can Do, What Bold Cannot Do, Contact Details

### Page Sections

1. **Header**
   - Title: "Property Management"
   - Subtitle: "Understanding Bold Property Management's Role"
   - Last updated date

2. **Overview Card**
   - Bold Property Management / Bold Solutions name
   - Role as day-to-day managing agent
   - Accountability statement (highlighted)

3. **Delegated Authority Section**
   - Uses DelegatedAuthority component (Story 10.2)
   - Lists what Bold CAN do

4. **Authority Limitations Section**
   - Uses AuthorityLimits component (Story 10.3)
   - Lists what Bold CANNOT do

5. **Contact Details Section**
   - Uses ManagementContacts component (Story 10.4)
   - Specific email addresses with guidance

6. **Records Custody Section**
   - Uses RecordsCustody component (Story 10.5)
   - What records Bold maintains

7. **Service Gaps Section** (optional, can be on transparency page)
   - Uses ServiceGaps component (Story 10.6)
   - What documentation is missing

8. **Related Links**
   - Link to /contacts
   - Link to /transparency
   - Link to /governance
