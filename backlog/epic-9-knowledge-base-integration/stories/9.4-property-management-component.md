# Story 9.4: Property Management Details Component

## User Story

**As a** homeowner needing to interact with property management
**I want** detailed information about Bold Property Management
**So that** I know their responsibilities, how to contact them, and what to expect

## Description

Create a detailed component for displaying property management information including their full responsibilities, communication platforms, email domains, and observed communication patterns. This goes beyond basic contact info to help residents understand what to expect.

## Acceptance Criteria

- [ ] Company name and role displayed
- [ ] Full mailing address shown
- [ ] Known email domains listed (@boldsolutions.net, @boldpropertymanagement.mailer.appfolio.us)
- [ ] Communication platform noted (AppFolio)
- [ ] All 10 observed responsibilities listed
- [ ] Historical context provided (previous management: Spaeth and Company)
- [ ] Component is reusable and can be placed on /contacts page
- [ ] Information matches Knowledge Base exactly

### Component

```typescript
// src/components/property-management-card.tsx

interface PropertyManagementCardProps {
  showResponsibilities?: boolean;
  showHistory?: boolean;
}

export function PropertyManagementCard({
  showResponsibilities = true,
  showHistory = false
}: PropertyManagementCardProps) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Property Management</CardTitle>
        <CardDescription>{PROPERTY_MANAGEMENT.role}</CardDescription>
      </CardHeader>
      <CardContent>
        <div className="space-y-4">
          <div>
            <h4>Bold Property Management / Bold Solutions</h4>
            <AddressDisplay address={PROPERTY_MANAGEMENT.mailingAddress} />
          </div>

          <div>
            <h4>Email Domains</h4>
            <ul>
              {PROPERTY_MANAGEMENT.emailDomains.map(domain => (
                <li key={domain}><code>{domain}</code></li>
              ))}
            </ul>
          </div>

          <div>
            <h4>Communication Platform</h4>
            <p>{PROPERTY_MANAGEMENT.platforms[0]}</p>
          </div>

          {showResponsibilities && (
            <div>
              <h4>Responsibilities</h4>
              <ul>
                {PROPERTY_MANAGEMENT.responsibilities.map(r => (
                  <li key={r}>{r}</li>
                ))}
              </ul>
            </div>
          )}

          {showHistory && (
            <div className="text-muted-foreground">
              <h4>Historical Note</h4>
              <p>Previous management: {HISTORICAL_MANAGEMENT.name}</p>
            </div>
          )}
        </div>
      </CardContent>
    </Card>
  );
}
```
