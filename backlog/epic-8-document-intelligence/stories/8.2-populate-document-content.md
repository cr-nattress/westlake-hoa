# Story 8.2: Populate Document Content

## User Story

**As a** user browsing documents
**I want** document records to contain the actual document content
**So that** I can read documents without downloading PDFs

## Description

Update the document data layer to use the content loader and populate the `content` field for all documents. Currently all documents have `content: null`.

## Acceptance Criteria

- [ ] All 16 MOCK_DOCUMENTS have content populated (where README exists)
- [ ] Content is loaded at build time (not runtime)
- [ ] Documents without README content gracefully fall back to summary
- [ ] No breaking changes to existing document queries
- [ ] Content is properly typed as `string | null`

## Technical Details

### File to Modify
`src/lib/data/documents.ts`

### Implementation Approach

**Option A: Static Content (Recommended for MVP)**
```typescript
import { loadDocumentContent } from './load-document-content';

const MOCK_DOCUMENTS: Document[] = [
  {
    id: "1",
    title: "Declaration of Condominium",
    slug: "declaration-of-condominium",
    type: "declaration",
    status: "current",
    content: loadDocumentContent("/docs/DeclarationofCondominium.pdf"),
    summary: "The original Declaration...",
    file_url: "/docs/DeclarationofCondominium.pdf",
    // ... rest unchanged
  },
  // ... update all documents
];
```

**Option B: Lazy Loading (Future Enhancement)**
```typescript
export async function getDocumentWithContent(slug: string): Promise<Document | null> {
  const doc = await getDocumentBySlug(slug);
  if (!doc) return null;

  if (!doc.content && doc.file_url) {
    doc.content = loadDocumentContent(doc.file_url);
  }

  return doc;
}
```
